Module 1: Document Structure

  • <!DOCTYPE> → Declares the document type
    • ↓ The <!DOCTYPE> declaration defines the document as HTML5.
    • ↓ It must be the very first thing in your HTML document, before the <html> tag.
    • ↓ It is not an HTML tag, but an instruction to the browser about what version of HTML the page is written in.
    • ↓ The correct declaration for HTML5 is <!DOCTYPE html> (not case-sensitive).
    • ↓ Browsers use this to ensure your page is rendered in standards mode, not quirks mode.
    • ↓ Omitting or misspelling <!DOCTYPE> can cause inconsistent display across browsers.
  • <html> → Root element
    • ↓ The <html> tag wraps all the content on the entire page.
    • ↓ It is the root of the HTML document tree.
    • ↓ The lang attribute specifies the language of the document (e.g., <html lang="en">).
    • ↓ All other HTML elements are descendants of <html>.
    • ↓ The <html> tag is required in every HTML document.
    • ↓ It usually contains two main sections: <head> and <body>.
  • <head> → Metadata container
    • ↓ The <head> tag contains meta-information about the document.
    • ↓ It is placed between the <html> and <body> tags.
    • ↓ Common elements inside <head> include <title>, <meta>, <link>, and <style>.
    • ↓ Content in <head> is not displayed on the page itself.
    • ↓ It helps browsers and search engines understand your page.
    • ↓ Only one <head> element is allowed per document.
  • <title> → Document title
    • ↓ The <title> tag sets the title of the web page (shown in the browser tab).
    • ↓ It must be placed inside the <head> section.
    • ↓ Only text (no HTML) is allowed inside <title>.
    • ↓ Every HTML document should have a <title> for accessibility and SEO.
    • ↓ The title is used as the default name when bookmarking the page.
    • ↓ There should be only one <title> per document.
  • <meta> → Metadata
    • ↓ The <meta> tag provides metadata about the HTML document.
    • ↓ It is always placed inside the <head> section.
    • ↓ Common uses: character set, page description, keywords, author, viewport settings.
    • ↓ Example: <meta charset="UTF-8"> sets the character encoding.
    • ↓ <meta name="viewport" content="width=device-width, initial-scale=1.0"> helps with responsive design.
    • ↓ <meta> tags are self-closing and do not have a closing tag.
  • <link> → External resources
    • ↓ The <link> tag defines a relationship between the current document and an external resource.
    • ↓ Most commonly used to link CSS files: <link rel="stylesheet" href="style.css">.
    • ↓ Placed inside the <head> section.
    • ↓ Can also be used for icons, prefetching, and alternate languages.
    • ↓ <link> is a self-closing tag and does not have a closing tag.
    • ↓ Multiple <link> tags can be used in one document.
  • <style> → Internal CSS
    • ↓ The <style> tag is used to define internal CSS styles for the document.
    • ↓ Placed inside the <head> section.
    • ↓ CSS rules are written between the opening and closing <style> tags.
    • ↓ Internal styles override external styles if they appear later in the cascade.
    • ↓ Use for small or page-specific styles; for larger projects, use external CSS files.
    • ↓ Only one <style> tag is needed per document, but multiple are allowed.
  • <body> → Page content
    • ↓ The <body> tag contains all the content of an HTML document (text, images, links, etc.).
    • ↓ Placed after the <head> section.
    • ↓ Only one <body> tag is allowed per document.
    • ↓ Content inside <body> is visible to users in the browser window.
    • ↓ Scripts and inline styles can also be placed inside <body>.
    • ↓ The <body> tag is required in every HTML document.
  • <script> → JavaScript
    • ↓ The <script> tag is used to embed or reference JavaScript code.
    • ↓ Can be placed in the <head> or <body> section.
    • ↓ Use the src attribute to link to an external JS file.
    • ↓ Inline scripts are written between the opening and closing <script> tags.
    • ↓ Scripts can manipulate the DOM, handle events, and add interactivity.
    • ↓ For best performance, place scripts just before the closing </body> tag.
  • <base> → Base URL
    • ↓ The <base> tag specifies the base URL for all relative URLs in the document.
    • ↓ Placed inside the <head> section, before any other elements that use URLs.
    • ↓ Only one <base> tag is allowed per document.
    • ↓ Example: <base href="https://www.example.com/">.
    • ↓ Useful for setting a consistent root for links and resources.
    • ↓ If not set, relative URLs are resolved relative to the document's location.

Module 2: Text & Headings

  • <h1> to <h6> → Headings
    • ↓ <h1> to <h6> define headings, from most important (<h1>) to least (<h6>).
    • ↓ Headings help structure your content and improve accessibility and SEO.
    • ↓ Only one <h1> should be used per page for the main title.
    • ↓ Headings are block-level elements and appear on their own line.
    • ↓ Browsers display headings in larger, bold text by default.
    • ↓ Use headings to create a logical outline of your content.
  • <p> → Paragraph
    • ↓ The <p> tag defines a paragraph of text.
    • ↓ Browsers automatically add space before and after each paragraph.
    • ↓ Paragraphs are block-level elements.
    • ↓ Use <br> for line breaks within a paragraph, not for separating paragraphs.
    • ↓ Paragraphs improve readability by breaking text into manageable chunks.
    • ↓ Always close paragraphs with </p>.
  • <br> → Line break
    • ↓ The <br> tag inserts a single line break.
    • ↓ It is an empty tag and does not require a closing tag.
    • ↓ Use <br> to break lines of text, such as in addresses or poems.
    • ↓ Avoid using <br> for layout or spacing; use CSS instead.
    • ↓ Too many <br> tags can make your code hard to maintain.
    • ↓ Example: Line 1<br>Line 2 displays on two lines.
  • <hr> → Horizontal rule
    • ↓ The <hr> tag creates a thematic break or horizontal line.
    • ↓ It is a self-closing tag and does not require a closing tag.
    • ↓ Use <hr> to separate content or sections visually.
    • ↓ Browsers render <hr> as a horizontal line across the page.
    • ↓ You can style <hr> with CSS for thickness, color, and width.
    • ↓ Avoid overusing <hr> for design; use it for logical breaks.
  • <strong> → Important text
    • ↓ The <strong> tag indicates that its contents are of strong importance.
    • ↓ Browsers typically render <strong> text as bold.
    • ↓ Use <strong> for text that needs emphasis for accessibility.
    • ↓ Screen readers announce <strong> text with emphasis.
    • ↓ Do not use <strong> for styling only; use CSS for visual bolding.
    • ↓ <strong> is a semantic tag, conveying meaning to browsers and assistive tech.
  • <em> → Emphasized text
    • ↓ The <em> tag marks text that should be emphasized.
    • ↓ Browsers typically render <em> text as italic.
    • ↓ Use <em> for words or phrases that need stress or emphasis.
    • ↓ Screen readers pronounce <em> text with emphasis.
    • ↓ Do not use <em> for styling only; use CSS for visual italics.
    • ↓ <em> is a semantic tag, conveying meaning to browsers and assistive tech.
  • <mark> → Highlighted text
    • ↓ The <mark> tag highlights text for reference or emphasis.
    • ↓ Browsers render <mark> text with a yellow background by default.
    • ↓ Use <mark> to draw attention to key words or search results.
    • ↓ <mark> is inline and can be used inside paragraphs or headings.
    • ↓ You can style <mark> with CSS for different highlight colors.
    • ↓ <mark> improves accessibility by indicating highlighted content.
  • <small> → Smaller text
    • ↓ The <small> tag renders text in a smaller font size.
    • ↓ Often used for disclaimers, copyright, or side comments.
    • ↓ <small> is inline and can be used inside paragraphs or headings.
    • ↓ Browsers display <small> text at about 80% of the parent size.
    • ↓ Use <small> for less important or fine print text.
    • ↓ You can style <small> with CSS for custom sizing.
  • <del> → Deleted text
    • ↓ The <del> tag shows text that has been deleted from a document.
    • ↓ Browsers render <del> text with a strikethrough line.
    • ↓ Useful for showing edits or changes in content.
    • ↓ <del> is inline and can be used inside paragraphs or lists.
    • ↓ You can pair <del> with <ins> for before/after edits.
    • ↓ <del> can have a datetime attribute for when the change occurred.
  • <ins> → Inserted text
    • ↓ The <ins> tag shows text that has been inserted into a document.
    • ↓ Browsers render <ins> text with an underline.
    • ↓ Useful for showing edits or additions in content.
    • ↓ <ins> is inline and can be used inside paragraphs or lists.
    • ↓ You can pair <ins> with <del> for before/after edits.
    • ↓ <ins> can have a datetime attribute for when the change occurred.
  • <sub> → Subscript
    • ↓ The <sub> tag displays text as subscript (below the baseline).
    • ↓ Commonly used for chemical formulas (e.g., H2O).
    • ↓ <sub> is inline and can be used inside paragraphs or headings.
    • ↓ Browsers render <sub> text smaller and lower than surrounding text.
    • ↓ Use for mathematical or scientific notation.
    • ↓ Always close with </sub>.
  • <sup> → Superscript
    • ↓ The <sup> tag displays text as superscript (above the baseline).
    • ↓ Commonly used for exponents (e.g., x2).
    • ↓ <sup> is inline and can be used inside paragraphs or headings.
    • ↓ Browsers render <sup> text smaller and higher than surrounding text.
    • ↓ Use for mathematical or scientific notation.
    • ↓ Always close with </sup>.

Module 3: Links, Images & Media

  • <a> → Hyperlink
    • ↓ The <a> tag defines a hyperlink, used to link from one page to another.
    • ↓ The href attribute specifies the URL of the page the link goes to.
    • ↓ Links can open in the same or a new tab using the target attribute.
    • ↓ You can link to external sites, internal pages, or specific sections using anchors.
    • ↓ Links are inline elements and can contain text, images, or other HTML.
    • ↓ Use descriptive link text for accessibility and SEO.
  • <img> → Image
    • ↓ The <img> tag embeds an image in the page.
    • ↓ The src attribute specifies the image file path or URL.
    • ↓ The alt attribute provides alternative text for accessibility.
    • ↓ Images are inline elements and do not require a closing tag.
    • ↓ You can set width and height using attributes or CSS.
    • ↓ Always use alt text for better SEO and accessibility.
  • <figure> → Figure with caption
    • ↓ The <figure> tag groups media content with a caption.
    • ↓ Commonly used with images, diagrams, or code snippets.
    • ↓ Use <figcaption> inside <figure> for the caption.
    • ↓ Helps associate captions with their related content semantically.
    • ↓ Improves accessibility and document structure.
    • ↓ <figure> is a block-level element.
  • <figcaption> → Figure caption
    • ↓ The <figcaption> tag provides a caption for a <figure> element.
    • ↓ Place <figcaption> as the first or last child of <figure>.
    • ↓ Used to describe or explain the content of the figure.
    • ↓ Improves accessibility and understanding for users and search engines.
    • ↓ Only one <figcaption> is allowed per <figure>.
    • ↓ <figcaption> is a block-level element.
  • <audio> → Audio content
    • ↓ The <audio> tag embeds sound content in documents.
    • ↓ Use the src attribute or <source> tags for audio files.
    • ↓ The controls attribute adds play, pause, and volume controls.
    • ↓ Supports formats like MP3, WAV, and OGG.
    • ↓ You can provide fallback text for unsupported browsers.
    • ↓ <audio> is a block-level element by default.
  • <video> → Video content
    • ↓ The <video> tag embeds video content in documents.
    • ↓ Use the src attribute or <source> tags for video files.
    • ↓ The controls attribute adds play, pause, and volume controls.
    • ↓ Supports formats like MP4, WebM, and OGG.
    • ↓ You can set width and height attributes or use CSS.
    • ↓ Provide fallback text for unsupported browsers.
  • <source> → Media source
    • ↓ The <source> tag specifies multiple media resources for <audio> or <video>.
    • ↓ Browsers use the first recognized format.
    • ↓ Place <source> tags inside <audio> or <video>.
    • ↓ Use the type attribute to specify the media type.
    • ↓ <source> is a self-closing tag.
    • ↓ Improves compatibility across different browsers.
  • <track> → Text tracks for media
    • ↓ The <track> tag adds subtitles, captions, or other text tracks to media.
    • ↓ Used inside <audio> or <video> elements.
    • ↓ The kind attribute specifies the type (subtitles, captions, etc.).
    • ↓ <track> is a self-closing tag.
    • ↓ Improves accessibility for users with hearing impairments.
    • ↓ Multiple <track> tags can be used for different languages or types.
  • <map> → Image map
    • ↓ The <map> tag defines an image map (clickable areas on an image).
    • ↓ Use the name attribute to reference the map from an <img> tag.
    • ↓ Contains one or more <area> tags that define clickable regions.
    • ↓ Useful for interactive diagrams or navigation images.
    • ↓ <map> is not visible on the page.
    • ↓ Each <area> defines a shape and coordinates.
  • <area> → Area in image map
    • ↓ The <area> tag defines a clickable area inside an image map.
    • ↓ Use the shape and coords attributes to define the region.
    • ↓ The href attribute specifies the link for the area.
    • ↓ <area> is a self-closing tag.
    • ↓ Multiple <area> tags can be used in one map.
    • ↓ Improves interactivity for images and diagrams.
  • <embed> → Embedded content
    • ↓ The <embed> tag embeds external content, such as PDFs or Flash, into a page.
    • ↓ Use the src attribute to specify the file to embed.
    • ↓ The type attribute specifies the media type.
    • ↓ <embed> is a self-closing tag.
    • ↓ You can set width and height attributes or use CSS.
    • ↓ Use sparingly, as not all browsers support all embedded content types.
  • <object> → External resource
    • ↓ The <object> tag embeds external resources, such as images, PDFs, or multimedia.
    • ↓ Use the data attribute to specify the file to embed.
    • ↓ The type attribute specifies the media type.
    • ↓ <object> can contain fallback content for unsupported browsers.
    • ↓ You can set width and height attributes or use CSS.
    • ↓ Useful for embedding interactive or multimedia content.
  • <iframe> → Inline frame
    • ↓ The <iframe> tag embeds another HTML page within the current page.
    • ↓ Use the src attribute to specify the URL of the page to embed.
    • ↓ You can set width and height attributes or use CSS.
    • ↓ <iframe> is a block-level element.
    • ↓ Useful for embedding videos, maps, or external content.
    • ↓ Use the title attribute for accessibility.

Module 4: Lists & Tables

  • <ul> → Unordered list
    • The <ul> tag creates an unordered (bulleted) list.
    • Contains <li> elements for list items.
    • Displays bullet points by default.
    • Used where order doesn’t matter.
    • Can style bullets using CSS.
    • Block-level element.
  • <ol> → Ordered list
    • Creates an ordered (numbered) list.
    • Contains <li> items.
    • Displays numbers/letters by default.
    • Use when order matters.
    • Custom styles with CSS or type.
    • Block-level element.
  • <li> → List item
    • Defines an item in <ul>, <ol>, or <menu>.
    • Can contain any HTML elements.
    • Can nest lists.
    • Block-level element.
  • <dl> → Description list
    • Defines a description list.
    • Uses <dt> and <dd>.
    • Useful for terms and descriptions.
  • <dt> → Term
    • Defines a term in a description list.
    • Must be inside <dl>.
  • <dd> → Description
    • Defines a description in <dl>.
    • Follows <dt>.
  • <table> → Table
    • Defines a table element.
    • Contains rows, headers, and cells.
    • Use only for tabular data.
  • <tr> → Table row
    • Defines a row inside a table.
    • Contains <td> or <th>.
  • <td> → Table data cell
    • Defines a cell for data.
    • Inside <tr>.
  • <th> → Table header cell
    • Defines a header cell.
    • Bold and centered by default.
  • <thead> → Table head
    • Groups header content.
    • Goes before <tbody>.
  • <tbody> → Table body
    • Groups main rows of the table.
  • <tfoot> → Table footer
    • Groups footer rows of the table.
  • <caption> → Table caption
    • Adds a title to the table.
    • Displayed above the table.
  • <colgroup> → Column group
    • Groups columns for styling.
    • Goes before <thead>.
  • <col> → Column
    • Defines a column in a <colgroup>.
    • Self-closing tag.

Module 5: Forms & Inputs

  • <form> → Form container
    • The <form> tag creates an HTML form for user input.
    • Forms can contain input fields, buttons, and other interactive elements.
    • Use the action attribute to specify where to send form data.
    • The method attribute defines how to send data (GET or POST).
    • Forms are block-level elements.
    • Always close the <form> tag.
  • <input> → Input field
    • The <input> tag creates interactive controls in a form.
    • Use the type attribute (text, password, email, etc.).
    • <input> is a self-closing tag.
    • Inputs can have attributes like name, placeholder, and required.
    • Inputs are inline elements by default.
    • Always use a <label> for accessibility.
  • <label> → Input label
    • Defines a label for an <input> element.
    • Use the for attribute to bind with input’s id.
    • Improves accessibility and usability.
    • Clicking the label focuses the input.
    • Labels can wrap the input directly.
  • <textarea> → Multi-line input
    • Creates a multi-line text input field.
    • Use rows and cols to set dimensions.
    • Place default text between opening and closing tags.
    • Useful for messages or comments.
  • <button> → Button
    • Creates a clickable button.
    • Use type: submit, reset, or button.
    • Can include text, icons, or HTML content.
  • <select> → Dropdown
    • Creates a dropdown list.
    • Contains <option> elements.
    • Use multiple to allow multiple selections.
  • <option> → Dropdown option
    • Defines an item in a dropdown.
    • Use value to assign a backend value.
    • Use selected to make default selection.
  • <optgroup> → Grouped options
    • Groups related <option> tags inside <select>.
    • Use label attribute to name the group.
  • <fieldset> → Group fields
    • Groups related form fields.
    • Renders with a border around the group.
    • Improves form structure.
  • <legend> → Fieldset caption
    • Provides a title for a <fieldset>.
    • Appears on the border of the fieldset.
  • <datalist> → Suggested options
    • Defines options for <input>.
    • Refer using the list attribute on input.
    • Contains <option> tags inside.
  • <output> → Display result
    • Shows result of calculation or input.
    • Inline element by default.
  • <progress> → Progress bar
    • Represents progress of a task.
    • Use value and max to define progress.
  • <meter> → Gauge/level meter
    • Displays scalar measurement within a known range.
    • Use for ratings, scores, disk usage, etc.
